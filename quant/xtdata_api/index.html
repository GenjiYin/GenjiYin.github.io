xtquant.xtdata 实战速查：MiniQMT 数据接口全解

整理了一份 xtquant.xtdata 的实战速查手册，覆盖行情获取、板块管理、财务数据、交易日历等高频接口，附常用别名与返回格式速记。

一、xtquant.xtdata 是什么

MiniQMT 是迅投提供的轻量化量化交易平台，可以在本地 Python 环境中直接调用行情和交易接口。而 xtquant.xtdata 就是负责「取数据」的模块——行情、财务、板块、日历、Level2 等数据都从这里出。

这篇文章不是官方文档的翻译，而是我从日常研究和实盘开发中整理出来的高频接口速查。核心目标只有一个：当你需要某种数据时，能在一分钟内定位到正确的函数、搞清楚参数含义、写出能跑的代码。

使用前提：已安装 MiniQMT 客户端并保持登录，Python 环境中已安装 xtquant 包。接口调用前通常需要先下载历史数据到本地缓存。

二、行情数据：最常用的一组接口

2.1 获取历史 K 线 / 分笔数据

from xtquant import xtdata

# 获取日线 K 线数据
data = xtdata.get_market_data(
    field_list=['time', 'open', 'high', 'low', 'close', 'volume', 'amount'],
    stock_list=['000001.SZ', '600000.SH'],
    period='1d',
    start_time='20240101',
    end_time='20241231',
    count=-1,
    dividend_type='front',
    fill_data=True
)

get_market_data 是日常用量最大的接口。几个关键参数的理解直接影响取数效率：

参数类型说明
field_listlist[str]字段列表，[] 表示全部字段
stock_listlist[str]股票代码列表，格式为 代码.市场
periodstrtick（分笔）、1m / 5m（分钟）、1d（日线）
countint-1 取全部，n 从结束时间向前取 n 条
dividend_typestr复权类型：none（不复权）、front（前复权）、back（后复权）
fill_databoolK 线缺失时是否填充（True 以前收盘填充，False 填 NaN）

返回格式是很多人踩坑的地方：

  • K 线周期（1m/5m/1d）：返回 {field: pd.DataFrame}，DataFrame 的 index 是股票代码，columns 是时间戳
  • 分笔周期（tick）：返回 {stock: np.ndarray}，数组按 time 升序排列

2.2 扩展版：get_market_data_ex

# 扩展版返回格式更直观：{stock: pd.DataFrame}
data_ex = xtdata.get_market_data_ex(
    field_list=['open', 'high', 'low', 'close'],
    stock_list=['000001.SZ'],
    period='1d',
    start_time='20240101',
    end_time='20241231'
)
# data_ex['000001.SZ'] 直接拿到该股票的 DataFrame

参数与 get_market_data 完全一致，区别在于 K 线数据返回 {stock: pd.DataFrame} 格式，每只股票的 OHLCV 集中在一张表里，更符合直觉。

2.3 实时盘口快照

# 获取最新全量盘口
tick = xtdata.get_full_tick(['000001.SZ', '600000.SH'])
# 返回：{'000001.SZ': {五档买卖、最新价、成交量...}}

get_full_tick 不做历史查询，只返回当前最新盘口，适合实时监控或盘中信号触发。

2.4 订阅与回调机制

# 订阅单只股票行情
seq = xtdata.subscribe_quote(
    stock_code='000001.SZ',
    period='1m',
    start_time='20250501093000',
    end_time='20250531150000',
    count=0,
    callback=on_data
)

# 订阅全推（整个市场）
seq2 = xtdata.subscribe_whole_quote(
    code_list=['SH', 'SZ'],
    callback=on_whole_data
)

# 取消订阅
xtdata.unsubscribe_quote(seq)

# 阻塞接收回调（通常在子线程中运行）
xtdata.run()

订阅机制适合盘中实时策略。注意 subscribe_whole_quote 的回调数据结构是 {stock1: data1, stock2: data2} 的扁平字典，和 subscribe_quote 的列表格式不同。

三、板块与合约：选股的第一步

做因子选股，第一步通常是确定股票池。xtdata 提供了完整的板块管理和合约信息接口。

3.1 获取板块列表与成份股

# 获取所有板块名称
sectors = xtdata.get_sector_list()

# 获取某板块的全部成份股
stocks = xtdata.get_stock_list_in_sector('沪深A股')
stocks = xtdata.get_stock_list_in_sector('中证500')
stocks = xtdata.get_stock_list_in_sector('自选股')  # 也支持自定义板块

# 添加自定义板块
xtdata.add_sector('我的策略池', ['000001.SZ', '600000.SH'])

# 删除自定义板块
xtdata.remove_sector('我的策略池')

get_sector_list 返回的是 MiniQMT 客户端左侧板块树中所有可见的板块名称，包括系统板块（行业、概念、指数）和用户自定义板块。这是构建股票池最方便的入口。

3.2 合约详情与类型判断

# 获取合约详细信息
detail = xtdata.get_instrument_detail('600000.SH')
# 包含：上市日期、涨跌停价、流通股本、总股本、最小变动单位、合约乘数等

# 判断证券类型
types = xtdata.get_instrument_type('000001.SZ')
# 返回：{'stock': True, 'index': False, 'fund': False, ...}

get_instrument_detail 返回的信息在做风控和参数计算时非常实用——比如根据 PriceTick 和 VolumeMultiple 计算最小变动金额，根据 UpStopPrice / DownStopPrice 判断涨跌停状态。

四、财务数据：基本面因子来源

# 获取财务数据
fin = xtdata.get_financial_data(
    stock_list=['000001.SZ', '600000.SH'],
    table_list=['Balance', 'Income', 'CashFlow'],
    start_time='20200101',
    end_time='20241231',
    report_type='report_time'  # report_time: 报表期; announce_time: 公告期
)
表名对应报表
Balance资产负债表
Income利润表
CashFlow现金流量表
Capital股本结构
HolderNum股东户数
Top10Holder十大股东
Top10FlowHolder十大流通股东
PershareIndex每股指标

返回格式：field、date、stock、value 四个列表，需要自行转换为宽表格式。report_type 的选择决定了时间对齐方式——report_time 按财报期对齐，announce_time 按公告日对齐，后者更适合回测避免未来信息泄露。

# 预下载财务数据（建议首次使用时先下载）
xtdata.download_financial_data(
    stock_list=['000001.SZ'],
    table_list=['Income', 'Balance'],
    start_time='20200101',
    end_time='20241231'
)

五、除权除息与复权

# 获取除权除息因子表
df = xtdata.get_divid_factors('000001.SZ', start_time='20200101', end_time='20241231')
# 返回 pd.DataFrame，包含分红、送股、配股等权息信息

当你需要自己做复权计算（比如自定义复权方式或验证数据一致性）时，这个接口是必需的。get_market_data 里的 dividend_type 参数已经内置了复权逻辑，日常用不到手动处理。

六、交易日历：容易被忽视但极其重要

很多策略的 bug 不是模型问题，而是「在非交易日调仓」「在交易时段外发单」这类低级错误。xtdata 提供了完整的日历接口。

# 获取交易日列表（返回毫秒时间戳列表）
dates = xtdata.get_trading_dates('SH', start_time='20240101', end_time='20241231')

# 获取节假日列表
holidays = xtdata.get_holidays()

# 获取交易日历（含日内时段）
calendar = xtdata.get_trading_calendar('SH', '20240101', '20241231', tradetimes=True)
# tradetimes=False: 返回日期列表
# tradetimes=True:  返回 {日期: [[开盘,收盘,类型], ...]}，类型 2=开盘竞价 3=连续交易 8=收盘竞价 9=盘后定价

# 获取指定股票的交易时段
times = xtdata.get_trade_times('600000.SH')
# 返回 [[开始时间, 结束时间, 交易类型], ...]

实战建议：在策略主循环开头，先用 get_trading_dates 校验当日是否为交易日；在定时任务中，用 get_trade_times 判断当前是否处于连续交易时段（类型 3），避免在竞价或盘后发单被拒。

七、数据下载：缓存是性能的关键

# 下载单只股票历史数据到本地缓存
ok = xtdata.download_history_data(
    stock_code='000001.SZ',
    period='1d',
    start_time='20200101',
    end_time='20241231'
)

# 下载板块数据
xtdata.download_sector_data()

# 下载可转债数据
xtdata.download_cb_data()

MiniQMT 的数据机制是「先下载，后查询」。第一次调用 get_market_data 取某只股票的历史数据时，如果本地没有缓存，接口会返回空或极慢。建议在策略启动时批量预下载股票池的历史数据，而不是每次现取现下。

八、Level2 数据

# Level2 实时行情（十档/千档）
l2_quote = xtdata.get_l2_quote(
    field_list=[], stock_code='000001.SZ',
    start_time='', end_time='', count=-1
)

# Level2 逐笔委托
l2_order = xtdata.get_l2_order(
    field_list=[], stock_code='000001.SZ',
    start_time='', end_time='', count=-1
)

# Level2 逐笔成交
l2_trans = xtdata.get_l2_transaction(
    field_list=[], stock_code='000001.SZ',
    start_time='', end_time='', count=-1
)

Level2 接口需要开通相应权限。逐笔委托和逐笔成交是 A 股市场粒度最细的数据，适合高频因子挖掘（如大单流入、委托簿失衡等）。

九、工具函数：时间格式转换

# 日期字符串 -> 毫秒时间戳
ts = xtdata.datetime_to_timetag('20240101150000', format='%Y%m%d%H%M%S')

# 毫秒时间戳 -> 日期字符串
dt = xtdata.timetag_to_datetime(1704096000000, '%Y%m%d')

xtdata 内部大量使用毫秒时间戳，而策略代码通常用字符串或 datetime 对象。这两个工具函数是数据类型转换的桥梁。

十一、小结

场景推荐接口
批量获取历史 K 线get_market_data_ex
获取最新盘口快照get_full_tick
盘中实时订阅subscribe_whole_quote
构建股票池get_sector_list + get_stock_list_in_sector
基本面因子get_financial_data
调仓日判断get_trading_dates
盘中时段判断get_trade_times
数据预下载download_history_data

xtquant.xtdata 的接口设计整体上是「功能全、参数多、返回格式不统一」。这篇速查的核心价值，是把散落在官方文档里的关键信息按实战场景重新组织，并标出那些真正会踩坑的地方——返回格式、时间格式、复权方式、预下载机制。

如果你也在用 MiniQMT 做研究，建议把这张表打印出来贴在显示器旁边。至少对我来说，它省下了不少翻文档的时间。
