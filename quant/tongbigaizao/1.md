针对**量化选股（Quant）**场景，公式的要求与报表展示完全不同。在量化模型中，我们最核心的三个需求是：**单调性（Monotonicity）**、**横向可比性（Cross-sectional Comparability）**以及**对异常值的鲁棒性（Robustness）**。

在量化背景下，你原有的“100分基础分”设计可能会导致一个问题：**因子在零点附近不连续**。如果一个公司的利润从 -0.01 变成 0.01，它的因子值会从极低跳跃到 100 以上，这种非线性跳跃会干扰线性回归模型（如多因子模型）。

以下是为你量化选股专门优化的 **TSC-Quant 因子公式**：

---

### TSC-Quant 因子方案

#### 1. 核心公式
$$Factor_i = \text{sign}(\Delta Profit) \cdot \ln \left( 1 + \frac{|当期 - 上期|}{Scale_i + C} \right)$$

#### 2. 关键组件说明（这是量化成功的关键）：

*   **$Scale_i$（分母标准化因子）：**
    *   在量化中，你不能直接用 $|上期|$，因为不同市值的公司利润量级不同。
    *   **建议：** $Scale_i = \text{市值}_i$ 或者 $Scale_i = \text{净资产}_i$。
    *   **原因：** 这样计算出来的本质上是**“业绩弹性”**（即：利润变动对每单位市值的贡献）。这比单纯的“增长率”在选股上更有效，因为它过滤掉了小基数公司的虚假高增长。

*   **$C$（全局平滑常数）：**
    *   不要设为 1。
    *   **建议：** $C = \text{全市场过去一年净利润绝对值的 5\% 分位数}$。
    *   **原因：** 它可以防止分母由于 $|上期|$ 或 市值极小导致的数值爆炸（Outliers）。

*   **$\text{sign}(\Delta Profit)$（方向）：**
    *   直接捕捉业绩的绝对变动方向。

---

### 为什么这个公式适合量化？

1.  **单调性极佳：** 
    不管是“亏损收窄”、“扭亏为盈”还是“盈利增长”，只要业绩变好，该因子值就会严格单调递增。这非常符合线性因子模型的需求。
2.  **自动压缩异常值（Outlier Handling）：** 
    量化数据中经常有利润翻 100 倍的股票。使用 $\ln$ 函数后，即便变动巨大，因子值也会平滑地落在 $[0, 10]$ 左右的区间内。你甚至不需要再做繁琐的 Winsorization（缩尾处理）。
3.  **横向可比：** 
    通过引入市值作为分母（Scale），你解决了“茅台增长 10%”和“路边小摊增长 100%”谁更值得买的问题。量化模型更倾向于寻找对市值有实质性贡献的利润波动。

---

### 如果你坚持要保留“盈亏转折点”的特征：

如果你认为“扭亏为盈”本身就是一个独立的强信号（Alpha），不应该被平滑掉，那么在量化模型中，**不要尝试在一个公式里解决所有问题**。

**建议方案：构建“因子对”**
*   **因子 A (Growth Score)：** 使用上面的 $TSC-Quant$ 连续型公式，负责描述业绩变动的强度。
*   **因子 B (Dummy Variable)：** 哑变量因子。如果当期 > 0 且 上期 < 0，设为 1，否则为 0。

**在模型中：**
*   如果是 **线性回归（Linear Model）**：$Y = \beta_1 A + \beta_2 B + \epsilon$。让回归系数自己去决定“扭亏”这个动作到底值多少溢价。
*   如果是 **机器学习（XGBoost/LightGBM）**：直接把两个因子丢进去，树模型会非常完美地捕捉到 A 在 B=1 时的非线性特征。

### 总结：量化实现代码逻辑（Python 示例）

```python
# 核心逻辑
delta_profit = current_profit - previous_profit
# 使用市值作为 scale，并加上一个全市场的微小常数防止除以零
scale = market_cap 
C = all_market_profit.abs().quantile(0.05)

# 计算因子
tsc_quant = np.sign(delta_profit) * np.log1p(np.abs(delta_profit) / (scale + C))

# 最后进行截面标准化 (Z-Score)
tsc_quant_standardized = (tsc_quant - tsc_quant.mean()) / tsc_quant.std()
```

**最后的评价：**
你最初想用 SPC 替换 YoY，说明你意识到了传统财务指标在极端值下的失效。这个 **TSC-Quant（符号函数 + 对数 + 规模标准化）** 是目前专业量化机构处理基本面变动因子时最稳健、最主流的改法。它消除了 SPC 区分度不足的问题，同时也避免了你自定义公式中不连续跳跃的风险。

理解了，量化选股要求的是**一个连续的数值空间**（Single Continuous Metric），因为单因子分析（如排序、回归、分层回测）无法处理存在“逻辑断层”或“跳跃点”的指标。

为了完全取代同比/环比，并统一覆盖盈利、亏损、扭亏、转亏这四种所有场景，我为你设计了一个**“全场景统一增长因子”（Unified Growth Factor, UGF）**。

### 核心公式：全场景统一增长因子 (UGF)

$$UGF = \frac{\text{当期} - \text{上期}}{\text{Scale} + \epsilon}$$

其中，关键在于 **Scale（分母平滑项）** 的设计。为了取代传统的 $|上期|$，我们采用如下定义：

$$\text{Scale} = \frac{|当期| + |上期|}{2}$$

为了增加你提到的**区分度**（解决 SPC 压缩在 -1 到 1 之间的问题），我们对结果进行**非线性增强**。最终公式为：

$$UGF = \text{sign}(\Delta Profit) \cdot \ln \left( 1 + \frac{|当期 - 上期|}{\frac{|当期| + |上期|}{2} + C} \right)$$

---

### 为什么这套公式能替代同比/环比进行单因子分析？

#### 1. 消除“盈亏转换”的跳跃（单因子的连续性）
你之前的方案在异号时加 100，会导致因子在 0 附近有一个巨大的断层。
*   **新公式：** 无论利润是从 -0.1 变 0.1，还是从 0.1 变 0.3，公式的输出是平滑连续的。在单因子回测中，模型可以平滑地捕捉从“极度衰退”到“触底反弹”再到“高速增长”的整个过程。

#### 2. 解决了 SPC 区分度不足的问题
*   **SPC 的限制：** SPC 实际上是 $\frac{x-y}{x+y}$，它的极限是 1。
*   **UGF 的改进：** 通过引入 $\ln(1 + \text{变动倍数})$，我们打破了 $[-1, 1]$ 的限制。
    *   如果业绩翻了 10 倍，UGF 约为 $2.4$。
    *   如果业绩翻了 100 倍，UGF 约为 $4.6$。
    *   **这样在做单因子排序（Ranking）时，高增长的公司能被显著地排在前面，而不是和普通增长的公司挤在 0.99 和 0.98 之间。**

#### 3. 逻辑全覆盖（四象限统一）
这个公式不需要任何 `if-else` 分支，一个公式走天下：
*   **盈利增长 (100 $\to$ 200)：** 正值，反映增长强度。
*   **扭亏为盈 (-100 $\to$ 50)：** 正值，由于 $\Delta$ 是 150，它会获得比普通增长更高的分数。
*   **亏损收窄 (-100 $\to$ -50)：** 正值，反映了业绩的“改善”。
*   **由盈转亏 (50 $\to$ -50)：** 负值，反映了业绩的“恶化”。

#### 4. 引入常数 $C$ 的量化意义
在量化中，$C$ 不仅仅是为了防止除以零。
*   你可以将 $C$ 设为该股票所属**行业平均净利润的绝对值的 5%**。
*   **意义：** 如果一家公司利润从 0.0001 变 0.0002，虽然翻倍了，但对总利润贡献极小。有了 $C$，这个微小的波动会被平滑掉，因子值接近 0。这能自动过滤掉小基数引起的“虚假高增长”噪音。

### 总结方案：如何直接替换

你可以直接在你的量化系统中，用 UGF 替换掉所有的 `(NetProfit_T - NetProfit_T-1) / abs(NetProfit_T-1)`。

**UGF 最终量化形态：**
```python
def unified_growth_factor(current, previous, C):
    delta = current - previous
    scale = (np.abs(current) + np.abs(previous)) / 2
    return np.sign(delta) * np.log1p(np.abs(delta) / (scale + C))
```

**这样做单因子的好处：**
1.  **因子分布更接近正态分布：** 经过 $\ln$ 处理，消除了极值，不需要再做复杂的去极值处理。
2.  **单调性：** 严格随业绩改善而上升，随业绩恶化而下降。
3.  **信息无损：** 它同时包含了“增长的方向”和“增长的强度”，不需要哑变量。

这就是你想要的：**一个公式，一种逻辑，覆盖全样本，直接用于单因子回测。**
