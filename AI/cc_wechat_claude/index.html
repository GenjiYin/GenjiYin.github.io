把 Claude Code 装进微信：零代码门槛的 cc-wechat 部署指南

一条 npx 命令、一次扫码登录，让你的手机微信变成 Claude Code 的远程遥控器。文字、图片、文件全支持，不需要懂逆向工程。

你有没有想过，不用守在电脑前，也能用手机微信直接指挥 Claude Code 帮你写代码、改 bug、查文档？

这不是科幻。一个叫 cc-wechat 的开源工具，把这件事变成了一条命令就能搞定的事。

这篇文章，我会带你从零开始，一步步把 Claude Code 桥接到微信——不需要你懂逆向工程，不需要折腾微信协议，扫码登录即可使用。

一、cc-wechat 是什么？一句话讲清楚

cc-wechat 是一个开源的"消息桥接器"。它的工作方式非常直白：

你在微信里发消息 → 腾讯 iLink API 转发 → 本地 Claude Code 处理 → 结果回传到微信

也就是说，你的手机微信变成了 Claude Code 的"远程遥控器"。文字、图片、文件都能发，Claude Code 在本地电脑上运行，处理完直接把结果发回微信。

不用 OpenClaw，不用破解微信，全部走腾讯官方的 iLink Bot API。安全、稳定、合法。

二、开始前，确认这三样东西

别急着动手，先花 30 秒确认你的环境是否满足要求：

项目 要求
Claude Code 版本 v2.1.80 及以上
Node.js 版本 >= 22
微信 iOS / Android / macOS / Windows 均可，支持 iLink Bot 扫码

Claude Code 版本怎么查？

打开终端，输入：

claude --version

如果显示的版本号低于 2.1.80，先升级 Claude Code 再往下看。

三、第一步：一键安装（真的就一条命令）

打开终端，复制粘贴下面这行命令，回车：

npx cc-wechat@latest install

这条命令会自动帮你做三件事：

  1. 把 cc-wechat 注册为 Claude Code 的 MCP Server
  2. 在终端弹出一个二维码，用微信扫码授权登录
  3. 告诉你下一步怎么启动

扫码成功后，你的登录信息会自动保存在本地（~/.claude/channels/wechat/ 目录下），下次不用再扫。

四、第二步：启动 Claude Code + 微信频道

安装完成后，启动命令是：

claude --dangerously-load-development-channels server:wechat-channel --dangerously-skip-permissions

启动后，Claude Code 会同时"监听"两个入口：

  • 你电脑上的终端（照常使用）
  • 你的微信消息

也就是说，你既可以在终端里打字用 Claude Code，也可以躺在沙发上用手机微信发消息给它。

为什么启动命令要加 --dangerously-skip-permissions？

Claude Code 在执行文件读写、命令运行等操作时，默认会弹出一个权限确认框，等你点"同意"才继续。

这本身是个安全设计，但用在微信远程场景就成了灾难——弹窗出现在电脑终端上，而你可能正在外面走路、吃饭、躺在床上。每聊一个需求都要跑回电脑前点一下确认，"远程遥控"的体验就彻底断了。

--dangerously-skip-permissions 的作用就是跳过这些弹窗，让 Claude Code 直接执行操作。对于通过微信互动 Claude Code 来说，这个参数几乎是必需的。

安全提醒：跳过权限意味着 Claude Code 会自主执行包括删除文件、运行 shell 命令在内的所有操作。建议只在本地开发环境使用，不要在生产服务器或包含敏感数据的环境中启用。

五、大概率会遇到的第一个坑："Channels are not currently available"

如果你启动后看到这句话，别慌。这不是你操作错了，而是 Anthropic 对 Channel 功能做了灰度发布，你的账号可能还没被放开。

解决方式同样是一条命令：

npx cc-channel-patch@latest

这个补丁工具会自动找到你电脑上的 Claude Code，修改三处限制：

  • 绕过 Anthropic 的灰度检查
  • 跳过 accessToken 验证（适合用代理的用户）
  • 移除"Channels are not currently available"的提示

它支持 Windows、macOS、Linux、WSL，还能自动识别你是原生安装还是 npm 安装的 Claude Code。打完补丁就能正常用了。

小提示：Claude Code 自动更新后，补丁可能会被覆盖。如果哪天突然又不能用了，重跑一次上面的命令即可。想还原的话，运行 npx cc-channel-patch unpatch。

六、手动安装方式（如果你想完全掌控每一步）

如果你不喜欢"一键安装"的魔法感，也可以分步来：

# 1. 全局安装 cc-wechat
npm i -g cc-wechat

# 2. 手动注册 MCP Server
claude mcp add -s user wechat-channel -- npx -y cc-wechat@latest

# 3. 扫码登录
npx cc-wechat login

# 4. 启动
claude --dangerously-load-development-channels server:wechat-channel

手动方式的好处是出了问题更容易排查——你知道每一步做了什么。

七、实际用起来是什么体验？

登录成功后，在微信里找到你授权的 iLink Bot 对话，直接发消息就行：

  • 发文字："帮我检查一下 src/auth/token.ts 里的刷新逻辑"
  • 发图片：截一张报错截图发过去，Claude Code 会读取并分析
  • 发文件：传 .log、.json 等文件，Claude Code 按需处理

Claude Code 收到消息后，会调用它所有的工具——读文件、改代码、运行命令、联网搜索——全部在你本地电脑上执行。处理完后，结果直接回传到微信。

因为我们启动时加了 --dangerously-skip-permissions，涉及文件修改、命令运行等操作时，Claude Code 会直接执行，不会再弹窗等你确认。这在微信远程场景下保证了流畅体验，但前提是你信任当前任务的安全性——它确实能"无脑"执行。

八、进阶：多账号配置（工作/生活分开）

cc-wechat 支持按项目使用不同的微信账号。比如个人微信用来玩，公司微信用来处理工作项目。

在项目根目录创建一个 .mcp.json 文件：

{
  "mcpServers": {
    "wechat-channel": {
      "command": "npx",
      "args": ["-y", "cc-wechat@latest"],
      "env": { "WECHAT_PROFILE": "work" }
    }
  }
}

每个 profile 的登录信息独立存储。不带 WECHAT_PROFILE 时用默认账号。典型用法：个人用默认，工作加 WECHAT_PROFILE=work。

九、六个常见踩坑点（提前帮你排雷）

1. 补丁被更新覆盖

Claude Code 自动更新后，cc-channel-patch 的补丁会失效。解法：更新后重跑一次补丁命令。

2. 登录态过期

iLink Bot 的 session 会过期。过期后重新扫码：npx cc-wechat login。

3. 语音消息只有文字

cc-wechat 对语音只提取文字转写，不传递原始音频。这是设计选择，Claude Code 本身也不处理音频。

4. 图片发过去没反应

确认你用的 Claude 模型支持 vision（多模态）。某些第三方中转的纯文本模型会忽略图片。

5. 必须先由微信端发第一条消息

iLink Bot 的机制要求用户先发起对话。Claude Code 不能主动推消息到微信——必须你先开口。

6. 安全问题

微信触发的操作在本地机器执行。建议：不要在生产服务器上运行；保持默认权限模式；如果多人能发消息给你，考虑限制白名单。

十、写在最后

cc-wechat 把"在微信里用 Claude Code"这件事压缩到了极致：一条 npx install、一次扫码、一个启动命令。它不重新包装 Claude Code，只做消息桥接——架构简单，出问题容易排查。

把它当成你已有 Claude Code 工作流的"微信遥控器"来用，而不是一个可以脱离终端独立运行的自动化系统，就不容易出事。

现在，拿起你的手机，发一条消息给你的 Claude Code 试试吧。
